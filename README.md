# Tender CRM Desktop

Десктоп-приложение для управления тендерами и работы с AI-поиском по документации.

## 🖥 Возможности

### Управление тендерами
- Создание, редактирование и архивирование тендеров
- Фильтрация по статусу (Новые, В работе, Выиграны, Проиграны, Архив)
- Поиск по названию, номеру извещения, заказчику
- Цветовая индикация статусов
- Счётчик связанных писем для каждого тендера

### Работа с письмами
- Просмотр всех обработанных писем
- Фильтрация по категориям: TENDER, SPAM, GENERAL, EMPTY
- Предпросмотр письма с метаданными
- Отображение тендерных данных, извлечённых LLM (НМЦК, сроки, название закупки)
- Просмотр вложений с размерами
- Связывание писем с тендерами через удобный диалог

### AI Поиск (RAG)
- Семантический поиск по документации тендеров
- Генерация ответов LLM с указанием источников
- Поиск в контексте конкретного тендера или по всем тендерам
- Настройка количества источников (top_k)
- История запросов с возможностью повторного использования
- Отображение процента сходства для каждого источника

### Настройки и статистика
- Проверка подключения к серверу
- Ручной запуск обработки почты
- Полная обработка ящика (за месяц, 3 месяца, год, весь ящик)
- Статистика в реальном времени: письма, тендеры, документы, чанки
- Автообновление статистики каждые 30 секунд

## 🚀 Сборка

### Вариант 1: Qt Creator (рекомендуется)

1. Откройте Qt Creator
2. **File → Open File or Project**
3. Выберите `CMakeLists.txt` из папки проекта
4. Выберите kit: **Desktop Qt 6.10.2 MinGW 64-bit** (или MSVC)
5. Нажмите **Configure Project**
6. **Build → Run** (или `Ctrl+R`)

### Вариант 2: CMake из командной строки

    # Создание папки сборки
    mkdir build && cd build

    # Конфигурация
    cmake .. -G "MinGW Makefiles"

    # Сборка
    cmake --build .

    # Запуск
    ./TenderCrmDesktop.exe

### Вариант 3: Сборка через Qt Creator CLI

    # Из папки проекта
    cmake -S . -B build -G "MinGW Makefiles"
    cmake --build build

## 🎯 Использование

### Первый запуск

1. Убедитесь, что запущен backend:
       cd tender-crm-backend
       python -m app.main

2. Запустите десктоп-приложение

3. Откройте вкладку **Настройки** и нажмите **Проверить** — должен появиться зелёный статус

4. Перейдите на вкладку **Тендеры** — должен отобразиться список

### Основные операции

| Действие | Как выполнить |
|----------|---------------|
| Обновить данные | `F5` или меню **Файл → Обновить** |
| Создать тендер | Вкладка "Тендеры" → кнопка **➕ Добавить** |
| Редактировать тендер | Двойной клик по строке |
| Обработать почту | Меню **Файл → Обработать почту** |
| Задать вопрос AI | Вкладка "AI Поиск" → ввести вопрос → **🔍 Задать вопрос** |
| Связать письмо с тендером | Вкладка "Письма" → кнопка **🔗 Связать с тендером** |

### Примеры запросов к AI

- "Какая НМЦК контракта?"
- "Какие сроки выполнения работ?"
- "Какие требования к участникам закупки?"
- "Кто заказчик?"
- "Какой размер обеспечения заявки?"

## 🏗 Архитектура

### Model-View паттерн

Приложение использует стандартный паттерн Qt Model/View:

- **Model** (`TendersModel`, `EmailsModel`) — отвечает за данные и их представление в таблице
- **View** (`QTableView`) — отображает данные и обрабатывает пользовательский ввод
- **Delegate** — кастомное отображение ячеек (цвета статусов, форматирование дат)

### Сетевое взаимодействие

Все данные получаются через `ApiClient` с использованием `QNetworkAccessManager`:

    Desktop App (Qt/C++)
         │
         │ HTTP/JSON
         ▼
    FastAPI Backend
         │
         ▼
    PostgreSQL + pgvector

### Асинхронность

HTTP-запросы выполняются асинхронно через сигналы/слоты Qt, что обеспечивает отзывчивый интерфейс даже при длительных операциях (обработка почты, RAG-запросы).

## 🔧 Настройка

### Изменение URL API

По умолчанию приложение подключается к `http://localhost:8000`. Изменить можно:

1. В коде: `src/api/ApiClient.cpp`, конструктор
2. Через вкладку **Настройки** (в разработке)

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `F5` | Обновить данные |
| `Ctrl+Q` | Выход из приложения |
| `Enter` (в поиске) | Выполнить поиск |
| Двойной клик | Редактировать тендер |

## 🐛 Отладка

### Логи в консоли

При запуске из Qt Creator в окне **Application Output** отображаются:
- SQL-запросы к API
- Ошибки сети
- Отладочная информация

### Распространённые проблемы

**Проблема**: "Не удалось подключиться к серверу"
**Решение**: убедитесь, что backend запущен на `http://localhost:8000`

**Проблема**: "Ошибка обработки писем"
**Решение**: проверьте логи backend, убедитесь, что IMAP настроен в `.env`

**Проблема**: "RAG-запрос не возвращает ответ"
**Решение**: убедитесь, что Ollama запущена и модель `intfloat/multilingual-e5-large` загружена
